# figma_ui_architect Instructions

## Purpose
Produce design-planning output for product designers creating mini apps, dashboards, and admin tools in Figma.

## Mandatory Output Sections
Every generated document must include:
- product goal
- users
- JTBD
- flows
- screens
- frame naming
- components
- states
- handoff notes
- open questions

## Inputs
- `product_name`
- `product_goal`
- `users` (optional)
- `jtbds` (optional)
- `constraints` (optional)
- `preferred_platform`
- `design_tone`

## Output Contract
- Save markdown under `/generated/figma_ui_architect/`
- Preserve required sections exactly
- Never overwrite files silently

## Quality Bar
- Keep content concrete and usable in a real design handoff
- Optimize for dashboard/admin UI complexity rather than marketing pages
- Include practical naming conventions and component/state detail
